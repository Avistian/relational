# Mission: Relational Deep Learning

## Why
Prove a contrarian bet: relational deep learning and foundational relational models are vastly undervalued today. Build the foundations year by year — from strong tabular baselines through cutting-edge research — so that when you deploy or publish results, you know exactly what you are standing on and can demonstrate that these models unlock value the current single-table paradigm misses.

## Success looks like
- Run rigorous baselines on relational benchmarks (starting with RelBench) and document where learned relational models beat manual feature engineering
- Reproduce and extend published RDL / foundation-model results with correct temporal splits and no leakage
- Complete the 6-year curriculum (240 lectures, 60+ core papers) with year-end exit exams
- Publish or ship a concrete result (benchmark leaderboard entry, open-source reproduction, or production deployment) that makes the undervaluation thesis legible to skeptics
- Explain the full stack — from XGBoost on a flat table to GNNs on a relational entity graph — without hand-waving
- Frame and execute **novel research** in Year 6 that tests the thesis with fair baselines and honest limitations

## Constraints
- **Baseline:** ~1 hour per day minimum, long-term commitment (**6-year expert curriculum** — see [CURRICULUM.md](./CURRICULUM.md)); extra time on strong days goes to labs, papers, or reproduction — never to skipping ahead of exit criteria
- Progress deliberately: basic tabular → advanced tabular → graph ML → RDL → foundation relational models → novel research
- Ground claims in high-trust sources and reproducible experiments, not hype
- Lessons and reference docs live in this workspace; daily practice mixes reading, interactive lessons, and hands-on coding

## Out of scope
- Chasing every new tabular architecture paper without finishing the current phase
- LLM-only approaches that ignore database structure (text serialization of tables is a tool, not the thesis)
- Production MLOps, cloud certification, or general data-engineering career content unless directly needed for an experiment
