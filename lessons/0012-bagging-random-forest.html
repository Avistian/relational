Bagging & Random Forest

Lesson 012 · Phase 1 · ~40 min

Lesson 011 showed a single tree carves feature space into axis-aligned rectangles. Grow that tree deep and it fits the training set almost perfectly — but it is high-variance: reshuffle a few rows and the splits jump around. This lesson's single skill: explain how averaging many decorrelated trees turns a high-variance learner into a stable one, and know what a Random Forest adds on top of plain bagging.

Bias vs variance, in one line. A deep tree has low bias, high variance. Bagging attacks the variance by averaging; boosting (Lesson 013) attacks the bias by correcting residuals. Same building block — the tree — two different failure modes to fix.

Bagging = Bootstrap AGGregatING

Train B trees, each on a bootstrap sample (draw n rows with replacement from the n training rows). Predict by averaging their probabilities (or voting). If the trees were perfectly independent, averaging B of them cuts variance by roughly 1/B while leaving bias unchanged (Breiman 1996). The catch: bootstrap trees are correlated, so the real reduction is smaller — which is exactly the gap Random Forest closes.

See variance collapse

Each faint gray line is one high-variance tree fit to a bootstrap sample; the bold blue line is their average; the dashed green line is the true function. Drag to add trees and watch the average steady toward the target while each individual tree stays jagged.

Random Forest = bagging + feature subsampling

Bagged trees are still correlated because the strongest feature tends to win the top split in every tree. Random Forest (Breiman 2001) breaks that correlation: at each split, only a random subset of features is considered (often √p for classification). Decorrelated trees average better, so the variance drop is larger than plain bagging for the same B.

  • Bootstrap rows → diversity across trees (bagging).
  • Random feature subset per split → decorrelation (the "random" in forest).
  • Average / vote → variance reduction without adding bias.

Out-of-bag (OOB) error — free validation

A bootstrap sample leaves out ~37% of rows on average ((1 − 1/n)n → e−1 ≈ 0.368). Each row is "out-of-bag" for the trees that never saw it, so you can score it with exactly those trees. The averaged OOB score is a nearly-free validation estimate — no separate fold needed. Treat it as a convenience, not a replacement for the leak-free CV spine you built in Q1.

Thesis bridge. Random Forest is a second honest baseline — but on tabular data the boosted GBDT you built in Q1 (and XGBoost in Lesson 014) usually beats RF, because boosting reduces bias where bagging only reduces variance (Grinsztajn 2022). RF is the control that proves the win in Lesson 013 is about boosting, not just "more trees."
