MLP & ResNet Tabular Baselines

Lesson 028 · Phase 1 · Q3 · ~50 min · core paper: Gorishniy, Rubachev, Khrulkov & Babenko 2021, §3.2

For four lessons we have been building the case that trees win on typical tabular data (L024 the benchmark; L025–L027 the three inductive biases). Today the argument turns a corner. If we are going to claim the relational bet beats a fair baseline, we have to know how to build the neural contestant properly — because half of the "deep learning loses on tables" folklore came from papers comparing against weak neural baselines. This lesson builds the two honest neural baselines everyone in the field now reports against: a plain MLP and a ResNet.

Its single skill: build the Gorishniy 2021 tabular ResNet — a linear embedding, a stack of pre-activation residual blocks (BatchNorm → Linear → ReLU → Dropout → Linear → Dropout, plus a skip connection), and a prediction head — explain why the skip connection is exactly what lets a deep tabular network train at all, and know that a properly-tuned MLP/ResNet (not the latest transformer) is the honest neural baseline a GBDT must be compared against.

One sentence. A deep stack of plain fully-connected layers gets harder to train as it gets deeper (the "degradation problem"); adding a skip connection — making each block output x + f(x) instead of f(x) — makes the identity map free, so depth stops hurting, and the resulting ResNet is a strong, honest baseline that is nonetheless still tied or beaten by a tuned GBDT on small tabular data.

First, the vocabulary (nothing here is assumed)

This is the first lesson where we build a neural network rather than use one as a foil, so it introduces the most machinery of any lesson so far. Every term is defined here, in plain language, before we rely on it. If you have met these before, skim; if not, read slowly — the rest of the lesson leans on all of them.

Baseline. A simple, well-understood model you compare a fancier model against. A result only means something relative to a baseline: "our model scores 0.80" is meaningless until you know a competent baseline scores 0.78 or 0.72. A large part of this lesson is about what counts as an honest baseline — one that is tuned well enough that beating it actually demonstrates something.
Linear (fully-connected) layer. The basic building block of these networks. It takes an input vector x of length d_in and produces an output vector of length d_out by a matrix multiply plus an offset: Linear(x) = W·x + b, where W is a d_out × d_in matrix of learned weights and b is a learned bias vector. "Fully-connected" means every output is a weighted sum of all the inputs. This is the same object as in L026 — and the reason an MLP is rotationally invariant: W·(Qx) = (WQ)·x, so a rotation of the input can be absorbed into W.
Activation function & ReLU. Between linear layers we apply a non-linear function elementwise; without it, a stack of linear layers would collapse into a single linear layer (a product of matrices is a matrix). The standard choice is the ReLU ("rectified linear unit"): ReLU(z) = max(0, z) — it passes positive values through unchanged and zeroes out negatives. That simple kink is what lets the network represent non-linear functions.
MLP (multilayer perceptron), hidden dimension. A multilayer perceptron is just a stack of linear layers with activations between them: Linear → ReLU → Linear → ReLU → … → Linear. The intermediate vector widths are the hidden dimensions (e.g. 256); the final layer's width is set by the task (one number for binary classification). This is the "MLP" that lost to trees in L025–L027 — but there we used it as a black box. Today we build it.
Training: epoch, minibatch, SGD/Adam. We fit the weights by gradient descent on a loss. One pass over the whole training set is an epoch; we do many. Each step uses a small random minibatch of rows (say 256) to estimate the gradient — this is stochastic gradient descent (SGD). Adam/AdamW is the standard adaptive optimizer that scales each weight's step by a running estimate of its gradient magnitude; it is what actually turns the knobs. You do not need the update equations today — just the vocabulary.
BatchNorm (batch normalization). A layer that, for each feature, subtracts the minibatch mean and divides by the minibatch standard deviation (then rescales by two learned parameters). It keeps the activations flowing through the network at a stable scale, which makes training faster and much less sensitive to initialization and learning rate. Crucially for today: because both our networks use BatchNorm, the deep one does not fail from "vanishing gradients" — we will need that fact to name the real culprit.
Dropout. A regularizer: during training it randomly sets a fraction p of activations to zero on each step (and rescales the rest), so the network cannot lean too hard on any one unit. At test time it is turned off. It reduces overfitting; it is not involved in the depth problem below (we set it to zero when isolating that).
Residual / skip connection. The idea at the heart of today. Instead of a block computing output = f(x), it computes output = x + f(x): the input is added back to the block's output via a "skip" (or "shortcut") that bypasses the layers. Then f only has to learn a residual — a small correction to x — rather than the whole transformation. Introduced for image networks by He et al. (2015); Gorishniy 2021 ports it to tables.
The degradation problem. The specific failure a skip connection fixes: as a plain deep network gets deeper, its accuracy gets worse — and, tellingly, its training accuracy gets worse too. That rules out overfitting (which would raise training accuracy while lowering test accuracy). The network simply becomes harder to optimize. He et al. named this the degradation problem and showed it is distinct from vanishing gradients. We will reproduce it directly.

The MLP baseline (Gorishniy §3.2)

Gorishniy et al. define the MLP baseline as a stack of identical blocks, each one a linear layer, a ReLU, and dropout, finished by a linear output layer:

MLPBlock(x) = Dropout(ReLU(Linear(x))) MLP(x) = Linear( MLPBlock( … MLPBlock(x) … ) )

That is the whole model — nothing exotic. The paper's point is that when this simple thing is tuned properly (width, depth, dropout, learning rate, weight decay), it is already a respectable tabular model, and several published "state-of-the-art" architectures did not reliably beat it. The trouble starts when you try to make it deep.

The problem: a deep plain MLP gets harder to train

Intuitively, more layers should never hurt: a deeper network contains the shallower one as a special case (set the extra layers to the identity and you are back where you started). So in the worst case a deep net should just match a shallow net. In practice it does not — plain deep MLPs get worse. The reason is subtle and worth stating precisely: the extra layers cannot easily learn to be the identity. A Linear → ReLU layer has to discover, from data, the specific weights that reproduce its input; that is a surprisingly awkward target for gradient descent, and every added layer multiplies the difficulty. The network is not overfitting — it literally cannot fit the training data as well. This is the degradation problem defined above.

Why it is not "vanishing gradients." You may have heard that deep nets fail because gradients shrink to zero on the way back to the early layers. That was the classic pre-2015 story, and BatchNorm largely fixes it — which is why both networks here use BatchNorm. The degradation we will see is a separate problem: even with healthy gradients, a deep plain stack is hard to optimize to a good solution. Keeping these two apart is the whole reason we measure training accuracy below.

The fix: the residual block

Gorishniy's ResNet block is the MLP block wrapped in a skip connection, with BatchNorm moved to the front (the "pre-activation" arrangement). Reading it inside-out:

ResNetBlock(x) = x + Dropout(Linear(Dropout(ReLU(Linear(BatchNorm(x))))))

The six operations in the middle are the residual branch f(x): normalize, project up to a wider hidden dimension, apply ReLU, drop, project back down to the block's width, drop again. Then — the one line that matters — the original x is added back. Here is the block laid out; toggle the skip connection on and off to see what changes (only one line) and what it means.

Why does adding x back solve the degradation problem? Because it makes the identity map free. If the best thing a block can do is "pass the input through unchanged," the residual branch just has to learn f(x) ≈ 0 — driving a few weights to zero, which gradient descent does easily — and the skip carries x through untouched. So a deeper residual network can always fall back to behaving like a shallower one; adding blocks can only help or do nothing, never actively hurt. (There is a gradient bonus too: the derivative of x + f(x) with respect to x is 1 + f′, so a clean "1" of gradient always flows back through the skip. But per the callout above, with BatchNorm the decisive effect here is the free identity, not gradient rescue.)

The decisive column is plain — train. If the plain net were overfitting, its training accuracy would stay pinned at 1.000 while only test accuracy fell. Instead its training accuracy also collapses, from 1.000 to 0.927 at depth 32 — the deep plain net cannot even memorize the training data. That is the degradation problem, caught red-handed: an optimization failure, not overfitting. The ResNet's training accuracy never leaves ~1.00, because the skip lets it keep the good shallow solution and add near-identity blocks on top.

The full architecture, and its two other pieces

We now have the block. The complete Gorishniy ResNet wraps a stack of them between an input embedding and a prediction head:

ResNet(x) = Head( ResNetBlock( … ResNetBlock( Linear_embed(x) ) … ) ) Head(x) = Linear( ReLU( BatchNorm(x) ) )
  • Input embedding (Linear_embed): a single linear layer that maps the raw feature vector up to the block width d_main. Every block preserves that width (so x + f(x) is well-defined — the two must have the same shape).
  • Prediction head: a final BatchNorm → ReLU → Linear that collapses the block width down to the output (one logit for binary classification). A logit is the raw pre-probability score; a sigmoid squashes it to a probability, and the loss (binary cross-entropy) is computed from it.

That is the entire model. Notice what it is not: there is no attention, no per-feature embedding, no tree. It is an MLP with skip connections and normalization — deliberately boring, so that beating it means something.

The honest bake-off: no universal winner

So how good is this neural baseline on real data? We ran the MLP, the ResNet, and a tuned GBDT (histogram gradient boosting, the L015 family) on credit_g — a small, categorical, irregular OpenML table, i.e. exactly the regime L024–L027 say favours trees — across five random splits.

ModelROC-AUC (mean ± sd)Accuracy (mean ± sd)Notes
MLP0.752 ± 0.0110.734 ± 0.002plain neural baseline
ResNet0.743 ± 0.0100.740 ± 0.013tied with the MLP (bands overlap)
GBDT0.793 ± 0.0250.754 ± 0.019strong, and far cheaper to train

Two honest readings, and you should hold both:

  • The GBDT wins here. On a 1000-row categorical table the tuned tree ensemble leads on both metrics — exactly what the Grinsztajn arc predicts, and a reminder that the ResNet does not repeal L024–L027. Building the neural baseline correctly did not make it win; it made the comparison fair.
  • The two neural baselines are tied. MLP (0.752) and ResNet (0.743) sit inside each other's error bars. On this small dataset the skip connection's payoff — which is about training deep networks — has little to bite on, because you do not need depth here. The ResNet earns its keep on larger, harder problems (and as the default block for the neural stacks of Year 2 and the RDL stack of Year 4).

This is the paper's real contribution, and the reason it is on our reading list: not a new winning architecture, but a piece of methodology. Gorishniy et al. showed that once you compare against a properly-tuned MLP and ResNet, a lot of the reported deep-learning progress on tables evaporates — several headline models did not beat these simple baselines. From now on, "our tabular DL model is SOTA" only counts if it clears these two bars first.

Explain it back

This is the load-bearing idea — put it in your own words before moving on.

Thesis bridge. This lesson is where the neural machinery the relational bet is built from finally appears. A relational deep-learning model is, at the end, encoder → message passing → a head — and that head, and often the row encoder, are exactly the residual-block MLPs of this lesson. Three consequences for the thesis. (1) The honest bar just got sharper. The single-table baseline an RDL result must beat is not a default MLP; it is a tuned GBDT and a tuned ResNet — Gorishniy's whole point is that weak neural baselines flatter new methods, and an RDL "win" over a weak baseline would be the same mistake. (2) The inductive-bias debts carry over. A ResNet is still an MLP-family model, so it still inherits the smoothness (L025), rotation (L026), and uninformative-feature (L027) biases; "add a skip connection" buys trainable depth, not tree-like robustness. (3) It is a foundation, not a detour. You cannot build or critique an RDL architecture without knowing what a residual block, BatchNorm, and dropout do — and Year 2 (FT-Transformer, TabR, TabM) and Year 4 (PyTorch Frame row encoders → GNN) assemble these very parts. Today you built the atom.

Where this is subtle (know this too)

  • Depth is a means, not an end. The skip connection's value is that it lets you go deep safely — but on small tabular data you rarely need to. The ResNet ties the MLP here precisely because 2 blocks are plenty. Do not reach for 32 layers on 1000 rows.
  • ResNet ≠ FT-Transformer. Gorishniy's paper also introduces the FT-Transformer, which uses per-feature embeddings and attention and tends to beat the ResNet on more datasets. We defer it to Year 2 (Q1) on purpose: the ResNet is the simplest honest neural baseline, and per-feature embeddings are the mechanism that starts to break the rotation/junk biases (a preview of how DL claws back the tree advantage).
  • "No universal winner" is the finding, not a hedge. Across the paper's full benchmark, GBDT wins some datasets and the DL models win others; ensembling both helps. Beware anyone who reports a single dataset and declares a category winner.
  • Tuning is doing the work. Just like the untuned-XGBoost story in L014, an untuned MLP is a genuinely weak baseline; the honest one is tuned (width, depth, dropout, learning rate, weight decay) under the L017 budget discipline and the L021 split discipline.
  • BatchNorm vs vanishing gradients. The reason we could pin the failure on the degradation problem (not vanishing gradients) is that both nets used BatchNorm. Remove BatchNorm and the plain net would fail even sooner, for the other reason too — a good example of why controlled ablations change one thing at a time (L022/L024).
