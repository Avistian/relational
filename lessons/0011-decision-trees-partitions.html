Decision Trees as Partitions

Lesson 011 · Phase 1 · ~40 min

Q1 built an honest sklearn baseline with HistGradientBoostingClassifier — a histogram-based tree ensemble. Before tuning XGBoost and LightGBM, you need to see what a single tree actually does: it carves the feature space into axis-aligned rectangles and assigns a constant prediction to each region.

This lesson's single skill: read a tree as a partition rule — one feature, one threshold, two child regions — and connect that geometry to the impurity reduction the algorithm maximizes.

From features to regions

A binary classification tree at depth 1 asks one question: 

if feature[j] ≤ threshold:
    predict majority class in left bucket
else:
    predict majority class in right bucket

Every split is axis-aligned: parallel to a feature axis. That is why trees excel at irregular, threshold-like interactions (Grinsztajn 2022) and struggle with smooth diagonal boundaries unless depth grows.

See one split

Two numeric features from a real credit dataset (OpenML German credit, Tier A). Points are colored by default vs bad credit; the dashed line is the depth-1 tree's split.

Gini impurity — what the split optimizes

At a node with class proportions p, Gini is G = 1 − Σ p_k². A split into left/right with proportions p_L, p_R and weights w_L, w_R (fraction of rows) yields: 

ΔG = G_parent − (w_L · G_left + w_R · G_right)

The learning algorithm searches features and thresholds to maximize ΔG (or an equivalent criterion). XGBoost later replaces Gini with a regularized gain on gradients — but the partition picture stays the same.
