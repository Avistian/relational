Q1 Checkpoint: A Reproducible Baseline

Lesson 010 · Phase 1 · ~45 min · capstone

Nine lessons, one habit: never let information leak from the future or the test set into training. This checkpoint is not a new concept — it is the moment you wire Lessons 001–009 into a single, reproducible, leak-free pipeline and evaluate it honestly. Everything after this — gradient boosting, neural nets, and ultimately relational deep learning — is measured against the baseline you build here.

This checkpoint's single skill: assemble the whole leakage spine into one Pipeline, evaluate it with the right metric against the prevalence baseline, and make the result reproducible.

The leakage spine, in one place

Each Q1 lesson contributed one vertebra. A competent baseline has all of them, and every fit-bearing step lives inside the pipeline so it only ever sees training folds:

The recipe

A reproducible sklearn baseline is one object you can fit and one number you can trust. Stateless engineering and every fit-bearing transform go inside a single Pipeline; a ColumnTransformer routes numeric and categorical columns to the right preprocessing; a fixed random_state and a pinned environment make the result repeatable. 

from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import StandardScaler, OneHotEncoder, FunctionTransformer
from sklearn.impute import SimpleImputer
from sklearn.ensemble import HistGradientBoostingClassifier
from sklearn.model_selection import StratifiedKFold, cross_val_score

pre = ColumnTransformer([
    ("num", SimpleImputer(strategy="median", add_indicator=True), num_cols),   # L006
    ("cat", OneHotEncoder(handle_unknown="ignore", sparse_output=False), cat_cols),
])
pipe = Pipeline([
    ("engineer", FunctionTransformer(add_ratio)),       # L009, stateless
    ("pre", pre),                                        # L005/L006, fit-bearing
    ("clf", HistGradientBoostingClassifier(random_state=0)),
])
cv = StratifiedKFold(5, shuffle=True, random_state=0)    # L003
score = cross_val_score(pipe, X_train, y_train, cv=cv,
                        scoring="average_precision").mean()   # L007/L008, PR-AUC

HistGradientBoostingClassifier is sklearn-native, handles NaNs without imputation, and is the strongest single-table baseline you can build with no extra dependencies — which is exactly why it's the yardstick. (Tuned XGBoost/LightGBM/CatBoost arrive in Q2.)

Always start with the dummy floor

Before any real model, fit a DummyClassifier(strategy="prior"). Its PR-AUC equals the prevalence and its ROC-AUC is 0.5 — the no-skill floor from Lesson 008. If a "fancy" model doesn't clearly beat the dummy, something is wrong (or the features are useless). Here is the verified bake-off on a realistic dataset — imbalanced, missing-laden, mixed-type, with an irregular signal:

Verified (sklearn 1.9, n=8000, 15.4% positive, 18% MAR-missing, fixed seed). HistGBDT wins on every metric. Both real models crush the dummy, and PR-AUC is read against the prevalence floor of 0.154 — "0.47" is ~3× the baseline. The tree wins because the signal is irregular axis-aligned interactions, the regime where trees beat linear and deep models (Grinsztajn et al. 2022).

Then calibrate the winner

Ranking quality is not trustworthy probabilities. Wrapping the winning HistGBDT in CalibratedClassifierCV(method="sigmoid", cv=5) lowered its Brier from 0.1078 → 0.1055 with ROC essentially unchanged — the Lesson 008 move, now part of a real baseline. If a deployed relational score will feed a threshold or an expected-value calculation, this step is not optional.

Reproducibility is part of the baseline

  • Fixed seeds everywhere randomness enters: split, CV, model (random_state=0).
  • Pinned environment — your .venv from requirements-labs.txt; record library versions in the output.
  • One callable — the whole baseline is a single Pipeline you can fit/predict, not a notebook of loose cells run out of order.
  • Stated prevalence next to every PR-AUC, so the baseline is always visible.
