Boosting Intuition

Lesson 013 · Phase 1 · ~40 min

Lesson 012 fixed a single tree's variance by averaging many decorrelated trees in parallel — a Random Forest. Boosting attacks the other half of the error. It grows trees one at a time, each one fitting what the current ensemble still gets wrong. This lesson's single skill: explain boosting as stagewise additive modelling that fits residuals to reduce bias, and know exactly how that differs from bagging.

Bagging vs boosting, in one line. Bagging trains trees independently and averages them to cut variance. Boosting trains trees sequentially, each correcting the last, to cut bias. Same building block — the tree — opposite failure mode.

The core idea: fit the residuals

Start with a trivial model — predict the mean for everyone. Look at what's left over: the residuals (truth − prediction). Now fit a small tree to those residuals, add a shrunken slice of it to your prediction, and recompute the residuals. Repeat. Each round the model spends its capacity exactly where the ensemble is still wrong, so the systematic error — the bias — shrinks round by round.

Friedman (2001) framed this precisely as gradient descent in function space: for squared-error loss the negative gradient is the residual, so "fit the residual" is literally "take a gradient step, using a tree as the step direction." Change the loss (to logistic loss for classification) and you fit the pseudo-residuals — the negative gradient of that loss — instead.

Gradient boosting (squared error), the skeleton: F0(x) = mean(y) for m = 1 … M:   ri = yi − Fm−1(xi)   ← residual (negative gradient)   fit a small tree hm to (xi, ri)   Fm(x) = Fm−1(x) + η · hm(x)   ← η = learning rate

Watch the bias fall

Each red bar is a residual — the gap the next stump will try to fit. The bold blue line is the running ensemble; the dashed green line is the true function. At M = 0 the model is a flat mean and the residuals are huge. Drag to add rounds and watch the blue curve bend toward the truth while the residual bars — and the MSE — collapse.

Why the learning rate matters

The learning rate η (shrinkage) scales each tree's contribution before it's added. Small η (0.01–0.1) takes tiny steps: it needs more rounds but generalizes better, because no single tree dominates. Large η can overshoot and overfit. Shrinkage plus "many weak learners" is the empirical heart of why boosting works — it's regularization by small steps. You will feel this trade-off directly in the lab.

Two families you'll hear named

  • AdaBoost (Freund & Schapire 1997) — the original: re-weight misclassified rows so the next tree focuses on them. It's a special case of the gradient view (exponential loss).
  • Gradient boosting (Friedman 2001) — the general recipe above; fit the negative gradient of any differentiable loss. XGBoost (Lesson 014) and LightGBM (Lesson 015) are fast, regularized implementations of this.

On our data

Verified with the relkit harness (5-fold PR-AUC). Two datasets tell the honest story — bagging and boosting each win where their failure mode dominates:

DatasetRandom ForestGradient BoostingWho wins
credit_g (n = 1,000, noisy) 0.901 ~0.88 RF — variance reduction on tiny data
adult (n = 48,842) 0.785 0.824 Boosting — bias reduction with real signal

This is the point, not a bug: on 1,000 noisy rows an untuned forest's variance reduction edges out untuned boosting, while on 48k rows with genuine structure boosting's bias reduction pulls clearly ahead. The boosting advantage grows further with tuning — which is exactly the job of XGBoost in Lesson 014. In the lab you'll build the residual loop by hand and reproduce the credit_g comparison so the near-tie is something you measured, not something you were told.

Thesis bridge. A competently-tuned gradient-boosted tree is the single-table baseline the relational thesis must beat fairly (Grinsztajn 2022). Understanding that boosting reduces bias by fitting residuals — not by magic — is what lets you later argue honestly that a relational model wins because of structure it can see and GBDTs cannot, not because your baseline was weak.
