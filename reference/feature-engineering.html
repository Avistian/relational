Feature Engineering

Reference · print-friendly cheat sheet

The core idea

A feature is a calculated field that makes a relationship the model needs learnable. Engineer the features a model cannot synthesize itself; skip the ones it can.

What models can / cannot learn (Heaton 2016)

Feature typeModels learn it alone?Engineer it?
log, polynomial, power, root (single column)yes (all 4 models)rarely worth it
ratio of differences \((a-b)/(c-d)\)no model couldyes — high value
interactions (A·B, effect of A depends on B)trees partially; linear noyes, esp. for linear models

Verified (\(y=(a-b)/(c-d)\), 5-fold R²): Ridge 0.644 → 0.998, HistGBDT 0.968 → 0.980 with the ratio engineered in.

Datetime: cyclical encoding

Integer hour 0…23 puts 23:00 and 00:00 far apart. Use sin/cos so the circle closes. Verified HistGBDT on an hour-of-day signal: raw timestamp R² 0.840 → 0.979 with sin/cos.

h = ts.dt.hour
X["hour_sin"] = np.sin(2*np.pi*h/24)
X["hour_cos"] = np.cos(2*np.pi*h/24)

Stateless vs fit-bearing (the leakage boundary)

TypeUsesExamplesWhere
Statelesswithin-row arithmetic onlyratios, differences, parsed hour, sin/coscannot leak; still keep in pipeline
Fit-bearingacross-row statisticstarget/mean encoding, scaling, KBins, imputationmust be a Pipeline step (fit on train folds)
 
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import TargetEncoder, FunctionTransformer

pipe = Pipeline([
    ("ratio",  FunctionTransformer(add_ratio)),   # stateless
    ("encode", TargetEncoder()),                  # fit-bearing: per-fold only
    ("model",  HistGradientBoostingClassifier()),
])
# Target-encoding the WHOLE frame leaks the label and inflates CV. Always inside the pipeline.

Relational features & the thesis spine

  • Most powerful relational features = aggregations over child rows (txn count, avg order value, days since last login).
  • Deep Feature Synthesis: follow foreign keys to a base table, stack aggregation + transform primitives; "depth" = relationships traversed (powers Featuretools).
  • Progression: manual FE → DFS (automated, you pick primitives) → RDL (GNN learns aggregations end-to-end). The hand-engineered baseline is what RDL must beat.
  • Always apply a point-in-time cutoff (Lesson 002): only aggregate child rows that existed before the prediction time.

Four mistakes

  1. Engineering what the model already learns (logs/polys of one column).
  2. Target encoding on the full dataset → label leak. Fix: per-fold pipeline step.
  3. Aggregating across the time cutoff → uses the future. Fix: PIT windows.
  4. Integer hour with a midnight cliff. Fix: sin/cos.
