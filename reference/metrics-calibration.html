Metrics: ROC vs PR & Calibration

Reference · print-friendly cheat sheet

Two separate questions

  1. Ranking: does it order positives above negatives? → ROC-AUC / PR-AUC.
  2. Calibration: can the probabilities be believed? → reliability curve + Brier.

A model can be a great ranker and badly calibrated. AUROC says nothing about calibration.

ROC vs PR

CurveAxesNo-skill baselineUse when
ROCTPR vs FPR0.5 alwaysranking quality, ~balanced classes
PRPrecision vs Recallprevalencerare positives, act on top of list (use MAP for ranking)

ROC is optimistic under imbalance (FPR denominator is true-negative-rich). Verified same-model: ROC-AUC 0.863 vs PR-AUC 0.720 at prevalence 0.177. Always state prevalence next to PR-AUC.

Calibration — reliability curve

Bin predictions; plot mean predicted vs observed frequency. On the diagonal = calibrated. RandomForest / boosted trees / SVMs bow off it (sigmoid distortion); logistic regression is usually close.

from sklearn.calibration import calibration_curve
frac_pos, mean_pred = calibration_curve(y_test, p, n_bins=10)   # plot frac_pos vs mean_pred

Brier score (one number, lower better)

Mean squared error of probability vs 0/1 outcome; a proper scoring rule (rewards ranking + calibration). Verified RF: raw 0.0898 → sigmoid 0.0856 → isotonic 0.0853 (AUROC unchanged).

from sklearn.metrics import brier_score_loss
brier_score_loss(y_test, p)

from sklearn.calibration import CalibratedClassifierCV
cal = CalibratedClassifierCV(base_estimator, method="sigmoid", cv=5).fit(X_train, y_train)
# cv= fits the calibrator on held-out folds -> no leakage (Lesson 005 instinct)

Relational / thesis notes

  • RelBench churn/fraud/recsys are rare-positive → report AUPRC / MAP + prevalence, not accuracy or bare AUROC.
  • A higher AUROC that loses on AUPRC at the operating recall is not a win — say so.
  • Deployed scores feeding a threshold or expected-value calc must be calibrated; GBDTs and GNNs are often not.

Decision flow

Evaluating a classifier?
 ├─ Ranking metric: balanced → ROC-AUC; rare positives → PR-AUC/MAP (state prevalence)
 └─ Will anyone ACT on the probability?
     ├─ no  → ranking metric is enough
     └─ yes → reliability curve + Brier; if off-diagonal, CalibratedClassifierCV(cv=...)
