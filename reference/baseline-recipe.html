Skeleton

from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import OneHotEncoder, FunctionTransformer
from sklearn.impute import SimpleImputer
from sklearn.ensemble import HistGradientBoostingClassifier
from sklearn.dummy import DummyClassifier
from sklearn.model_selection import train_test_split, StratifiedKFold, cross_val_score
from sklearn.calibration import CalibratedClassifierCV
from sklearn.metrics import average_precision_score, roc_auc_score, brier_score_loss

Xtr, Xte, ytr, yte = train_test_split(X, y, test_size=0.3, stratify=y, random_state=0)
prevalence = yte.mean()                       # PR-AUC no-skill floor

pre = ColumnTransformer([
    ("num", SimpleImputer(strategy="median", add_indicator=True), num_cols),
    ("cat", OneHotEncoder(handle_unknown="ignore", sparse_output=False), cat_cols),
])
pipe = Pipeline([
    ("engineer", FunctionTransformer(add_ratio)),     # stateless FE
    ("pre", pre),
    ("clf", HistGradientBoostingClassifier(random_state=0)),
])
cv = StratifiedKFold(5, shuffle=True, random_state=0)
cv_pr = cross_val_score(pipe, Xtr, ytr, cv=cv, scoring="average_precision").mean()

# always benchmark the floor
dummy_pr = DummyClassifier(strategy="prior").fit(Xtr, ytr)  # PR-AUC == prevalence, ROC == 0.5

# calibrate the winner if probabilities will be acted on
cal = CalibratedClassifierCV(pipe, method="sigmoid", cv=5).fit(Xtr, ytr)
p = cal.predict_proba(Xte)[:, 1]
print(average_precision_score(yte, p), roc_auc_score(yte, p), brier_score_loss(yte, p))

Checklist before you report a number

  • Did the dummy get beaten clearly? If not, stop and debug.
  • Is every fit-bearing step inside the Pipeline (nothing fit on full data)?
  • Is the metric PR-aware and stated next to the prevalence?
  • If probabilities will be acted on — is the winner calibrated (Brier reported)?
  • Seeds fixed, env pinned, versions printed?

Thesis note

This baseline is the yardstick. Every later result — XGBoost (Q2), neural tabular models, and RDL — is reported as better/worse than a competently-built, leak-free, calibrated GBDT baseline. Grinsztajn 2022 explains why that GBDT is hard to beat on single-table data. Q2 swaps HistGBDT for tuned XGBoost/LightGBM/CatBoost on this same harness.
