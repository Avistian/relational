Pipelines & Preprocessing

Reference · print-friendly cheat sheet

The one rule

Anything with a fit (scaler, imputer, encoder, feature selector, PCA) goes inside a Pipeline so it refits per CV fold on training rows only. Fitting before the split leaks test statistics into training and inflates the score.

What each step learns (and leaks)

StepLearns in fit
StandardScalerper-column mean & std
SimpleImputerper-column mean / median / mode
OneHotEncoderthe category set
SelectKBestfeatures correlated with y (worst leak)
PCAprincipal axes

Basic pipeline

from sklearn.pipeline import Pipeline, make_pipeline
from sklearn.impute import SimpleImputer
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression

pipe = Pipeline([
    ("impute", SimpleImputer(strategy="median")),
    ("scale",  StandardScaler()),
    ("clf",    LogisticRegression(max_iter=1000)),
])
scores = cross_val_score(pipe, X, y, cv=5)   # whole chain refit per fold

make_pipeline(...) auto-names steps lowercase; Pipeline lets you name them (needed for tuning).

Mixed columns: ColumnTransformer

from sklearn.compose import ColumnTransformer, make_column_selector
from sklearn.preprocessing import OneHotEncoder

numeric = Pipeline([("impute", SimpleImputer(strategy="median")),
                    ("scale",  StandardScaler())])
categorical = Pipeline([("impute", SimpleImputer(strategy="most_frequent")),
                        ("onehot", OneHotEncoder(handle_unknown="ignore",
                                                 sparse_output=False))])

pre = ColumnTransformer([
    ("num", numeric,     make_column_selector(dtype_include="number")),
    ("cat", categorical, make_column_selector(dtype_include=object)),
])
model = Pipeline([("pre", pre), ("clf", LogisticRegression(max_iter=1000))])

Two non-optional settings (verified sklearn 1.9)

SettingWhy
OneHotEncoder(handle_unknown="ignore")test-only categories crash otherwise; encodes unseen as all-zeros
sparse_output=Falserequired with set_output(transform="pandas") — sparse + pandas raises

Readable feature names

pre.set_output(transform="pandas")
# columns -> num__a, num__b, cat__cat_x, cat__cat_y   (step__feature)

Tune the whole pipeline (step__param)

from sklearn.model_selection import GridSearchCV
param_grid = {
    "pre__num__impute__strategy": ["median", "mean"],
    "clf__C": [0.1, 1.0, 10.0],
}
search = GridSearchCV(model, param_grid, cv=5)

Double underscore descends one level: step → substep → param. Composes with nested CV (Lesson 004) — wrap search in cross_val_score.
