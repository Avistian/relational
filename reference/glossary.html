Q1 · Leakage & evaluation spine (L001–L010)

TermDefinitionLesson
Single-table assumptionTabular learners consume a design matrix + label vector, not raw tables, so relational data must be flattened first. The thing the RDL thesis argues throws away signal.001
Design matrix (X)The feature matrix: one row per training example, one column per feature.002
Label vector (y)The target aligned row-for-row with X — what the model predicts.002
Temporal leakageAny feature that encodes information unavailable at the prediction cutoff time; makes offline scores fantasy.001
Point-in-time (PIT)Building each row's features using only events strictly before its cutoff timestamp — the structural defence against temporal leakage.002
Stratified splitA train/test split that preserves the class prevalence in each part.003
Cross-validation (CV)Repeatedly hold out a fold, train on the rest; average the held-out scores. Fit only on the training portion of each fold.003
Grouped CVCV where all rows of an entity (customer, patient) stay together in one fold, so the model can't memorise an entity seen in training.004
Nested CVAn inner CV selects hyperparameters; an outer CV scores the whole selection procedure. The honest estimate of tuned performance.004
best_score_A grid search's best validation score — optimistic, because it was chosen using that same validation data (selection bias). Not a training score.004
Pipeline (fit scope)A container that fits every data-learning step (scaler, encoder, selector) on the training portion of each fold — never on all data before splitting.005
MCAR / MAR / MNARMissingness mechanisms: Completely At Random; At Random given observed X; Not At Random (depends on the unobserved value itself).006
Missingness indicatorA binary "was-missing" feature; preserves the pattern that imputation alone erases (essential under MNAR).006
Class imbalanceA skewed positive rate; makes accuracy misleading (the accuracy paradox) and shifts the decision threshold.007
SMOTESynthetic minority oversampling; must run inside the CV fold or it leaks.007
ROC-AUCRanking quality via TPR vs FPR; no-skill baseline 0.5; optimistic under imbalance.008
PR-AUCPrecision vs recall; no-skill baseline = prevalence; the honest lens on rare positives.008
Calibration / BrierWhether predicted probabilities match observed frequencies (reliability curve); Brier = MSE of probability vs outcome. Calibrator fit on held-out data.008
Target encodingReplacing a category by its mean target; must be out-of-fold or it leaks the label.009
Leakage spineThe Q1 checklist of leakage sources (PIT, split, pipeline scope, resampling, encoding) a reproducible baseline must clear.010

Q2 · Trees, boosting, ensembling (L011–L020)

TermDefinitionLesson
Decision treeA learner that recursively partitions feature space with axis-aligned splits into piecewise-constant regions.011
Gini impurity1 − p² − (1−p)²: the chance two random draws from a node differ in label.011
Split gainImpurity drop from a split = parent impurity − weighted children impurity (regularized form subtracts a γ toll).011
BaggingBootstrap-aggregating: average many trees on resampled data; variance falls ~1/B.012
Random ForestBagging + random feature subsets per split; OOB score estimates test error for free.012
OOB (out-of-bag)Rows not in a tree's bootstrap sample; used to score that tree without a separate holdout.012
BoostingFit each new learner to the residuals/gradients of the current ensemble; a stagewise additive model.013
Shrinkage (learning rate η)Scale each boosting step down so more, smaller steps generalise better.013
Gradient / hessian (g, h)First/second derivatives of the loss w.r.t. the current model score F, recomputed each round.014
Regularized objective (XGBoost)Loss + Ω = γT + ½λ‖w‖²; penalises leaf count and leaf-weight magnitude.014
Leaf weight w*Closed-form optimum w* = −G / (H + λ); λ shrinks it toward zero.014
Histogram splitsBin features once, split on bins — the speedup that made modern GBDTs fast (XGBoost-hist, LightGBM).015
Leaf-wise growthGrow the best-gain leaf next (best-first), so num_leaves — not max_depth — is the primary capacity knob.015
GOSSGradient-based One-Side Sampling: keep large gradients, sample the rest, amplify the sample by (1−a)/b to stay unbiased.015
EFBExclusive Feature Bundling: pack mutually-exclusive sparse features into one, shrinking the feature count.015
Ordered target statisticsCatBoost's leak-free category encoding: encode row i from its permutation prefix + a prior, never its own label.016
Ordered boostingCompute a row's gradient from a model that never saw that row — kills prediction shift.016
Oblivious treeA symmetric tree using the same split across a whole level (a decision table); regularized and fast (CatBoost).016
Grid vs random searchAt equal budget under low effective dimensionality, random search tries ~n distinct values on the key axis vs a grid's √n — so it wins.017
Low effective dimensionalityOnly a few hyperparameters actually matter; the premise that makes random search beat grid.017
StackingA meta-learner trained on base-model predictions; must use out-of-fold preds or it leaks.018
Out-of-fold (OOF) meta-featuresBase predictions for a row made by models that didn't train on it — the honest input to a stack.018
BlendingSimple (often averaged) combination of base models; weaker than stacking but needs no meta-learner.018
Inductive biasThe kind of function a learner reaches for first: trees → axis-aligned piecewise-constant; MLP → smooth, rotation-invariant.019
Rotational invarianceLearning the same thing under a rotation of feature space; a virtue on images, a weakness on tables where columns are individually meaningful.019
Fair-comparison contractFix data, split, metric, tuning budget, and preprocessing scope up front; report the verdict + gap honestly. The reviewer stance from the Q2 checkpoint.020
Thesis vocabulary (used from Year 1, defined in full later). Relational deep learning (RDL) — learning directly over a database's entity graph rather than a flattened table. Relational entity graph (REG) — the graph whose nodes are rows and edges are foreign keys. Message passing — how a GNN propagates information along those edges. These get their own entries when their lessons land (Year 3+); listed here so usage stays consistent from the start.
