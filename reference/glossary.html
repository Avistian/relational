Q1 · Leakage & evaluation spine (L001–L010)

TermDefinitionLesson
Single-table assumptionTabular learners consume a design matrix + label vector, not raw tables, so relational data must be flattened first. The thing the RDL thesis argues throws away signal.001
Design matrix (X)The feature matrix: one row per training example, one column per feature.002
Label vector (y)The target aligned row-for-row with X — what the model predicts.002
Temporal leakageAny feature that encodes information unavailable at the prediction cutoff time; makes offline scores fantasy.001
Point-in-time (PIT)Building each row's features using only events strictly before its cutoff timestamp — the structural defence against temporal leakage.002
Stratified splitA train/test split that preserves the class prevalence in each part.003
Cross-validation (CV)Repeatedly hold out a fold, train on the rest; average the held-out scores. Fit only on the training portion of each fold.003
Grouped CVCV where all rows of an entity (customer, patient) stay together in one fold, so the model can't memorise an entity seen in training.004
Nested CVAn inner CV selects hyperparameters; an outer CV scores the whole selection procedure. The honest estimate of tuned performance.004
best_score_A grid search's best validation score — optimistic, because it was chosen using that same validation data (selection bias). Not a training score.004
Pipeline (fit scope)A container that fits every data-learning step (scaler, encoder, selector) on the training portion of each fold — never on all data before splitting.005
MCAR / MAR / MNARMissingness mechanisms: Completely At Random; At Random given observed X; Not At Random (depends on the unobserved value itself).006
Missingness indicatorA binary "was-missing" feature; preserves the pattern that imputation alone erases (essential under MNAR).006
Class imbalanceA skewed positive rate; makes accuracy misleading (the accuracy paradox) and shifts the decision threshold.007
SMOTESynthetic minority oversampling; must run inside the CV fold or it leaks.007
ROC-AUCRanking quality via TPR vs FPR; no-skill baseline 0.5; optimistic under imbalance.008
PR-AUCPrecision vs recall; no-skill baseline = prevalence; the honest lens on rare positives.008
Calibration / BrierWhether predicted probabilities match observed frequencies (reliability curve); Brier = MSE of probability vs outcome. Calibrator fit on held-out data.008
Target encodingReplacing a category by its mean target; must be out-of-fold or it leaks the label.009
Leakage spineThe Q1 checklist of leakage sources (PIT, split, pipeline scope, resampling, encoding) a reproducible baseline must clear.010

Q2 · Trees, boosting, ensembling (L011–L020)

TermDefinitionLesson
Decision treeA learner that recursively partitions feature space with axis-aligned splits into piecewise-constant regions.011
Gini impurity1 − p² − (1−p)²: the chance two random draws from a node differ in label.011
Split gainImpurity drop from a split = parent impurity − weighted children impurity (regularized form subtracts a γ toll).011
BaggingBootstrap-aggregating: average many trees on resampled data; variance falls ~1/B.012
Random ForestBagging + random feature subsets per split; OOB score estimates test error for free.012
OOB (out-of-bag)Rows not in a tree's bootstrap sample; used to score that tree without a separate holdout.012
BoostingFit each new learner to the residuals/gradients of the current ensemble; a stagewise additive model.013
Shrinkage (learning rate η)Scale each boosting step down so more, smaller steps generalise better.013
Gradient / hessian (g, h)First/second derivatives of the loss w.r.t. the current model score F, recomputed each round.014
Regularized objective (XGBoost)Loss + Ω = γT + ½λ‖w‖²; penalises leaf count and leaf-weight magnitude.014
Leaf weight w*Closed-form optimum w* = −G / (H + λ); λ shrinks it toward zero.014
Histogram splitsBin features once, split on bins — the speedup that made modern GBDTs fast (XGBoost-hist, LightGBM).015
Leaf-wise growthGrow the best-gain leaf next (best-first), so num_leaves — not max_depth — is the primary capacity knob.015
GOSSGradient-based One-Side Sampling: keep large gradients, sample the rest, amplify the sample by (1−a)/b to stay unbiased.015
EFBExclusive Feature Bundling: pack mutually-exclusive sparse features into one, shrinking the feature count.015
Ordered target statisticsCatBoost's leak-free category encoding: encode row i from its permutation prefix + a prior, never its own label.016
Ordered boostingCompute a row's gradient from a model that never saw that row — kills prediction shift.016
Oblivious treeA symmetric tree using the same split across a whole level (a decision table); regularized and fast (CatBoost).016
Grid vs random searchAt equal budget under low effective dimensionality, random search tries ~n distinct values on the key axis vs a grid's √n — so it wins.017
Low effective dimensionalityOnly a few hyperparameters actually matter; the premise that makes random search beat grid.017
StackingA meta-learner trained on base-model predictions; must use out-of-fold preds or it leaks.018
Out-of-fold (OOF) meta-featuresBase predictions for a row made by models that didn't train on it — the honest input to a stack.018
BlendingSimple (often averaged) combination of base models; weaker than stacking but needs no meta-learner.018
Inductive biasThe kind of function a learner reaches for first: trees → axis-aligned piecewise-constant; MLP → smooth, rotation-invariant.019
Rotational invarianceLearning the same thing under a rotation of feature space; a virtue on images, a weakness on tables where columns are individually meaningful.019
Fair-comparison contractFix data, split, metric, tuning budget, and preprocessing scope up front; report the verdict + gap honestly. The reviewer stance from the Q2 checkpoint.020

Q3 · Evaluation rigor & benchmark literacy (L021–L030)

TermDefinitionLesson
i.i.d. / exchangeable rowsThe assumption every shuffled split makes: rows are independent and identically distributed, so which rows land in test vs train does not matter. Broken when data is time-ordered and the world drifts.021
Temporal splitOrder rows by time and cut once: train on the past, test on the future — mirrors deployment. The default posture when a time order or drift exists.021
Concept driftThe feature→label rule changes over time (distinct from a shifting class balance). Makes a model trained on the past stale on the future.021
Covariate shiftThe input feature distribution changes over time while the rule may stay fixed; a second, separate kind of distribution shift.021
TimeSeriesSplitExpanding-window CV: each fold trains on an earlier block and validates on the next, never shuffled — the time-respecting analogue of KFold.021
Optimism gapThe amount a random split overstates deployment performance versus a temporal split (here random-CV 0.846 vs temporal 0.758 → +0.088).021
Data leakageAnything the model learns at training time that it could not legitimately know at prediction time; inflates the test score too, so CV looks fine. Kapoor & Narayanan's taxonomy: 8 types in 3 families.022
Illegitimate feature (L2)A column that proxies the label or would not exist at prediction time (computed from/after the outcome). Passes every pipeline/split check — caught only by auditing provenance.022
Non-independence (L3.2) / duplicates (L1.4)The same underlying record (patient, device) appears in both train and test, so the model half-memorises the answer; the unit of independence is the record, not the row. Fix: a grouped split.022
Reproducibility collapseWhen an apparent complex-vs-simple win vanishes once a leak is removed (demo: RF 0.935 vs LR 0.719 → 0.712 vs 0.721). A large gap is a leak hypothesis first.022
Model info sheetKapoor & Narayanan's prescribed defence: a structured form with a question per leak type, filled before publishing an ML claim, so leaks surface early.022
Paired significance testBecause two models are scored on the same folds, test the per-fold differences d_i = score_A(i) − score_B(i); pairing cancels fold-difficulty and is more powerful than comparing independent columns.023
Anticonservative testA test that rejects the null too often (too many false positives) because it underestimates variance — e.g. a paired t-test on correlated CV folds.023
Corrected resampled t-testNadeau & Bengio (2003): k-fold training sets overlap, so fold scores are correlated; inflate the variance of the mean by (1/n + n_test/n_train) — for k-fold, + 1/(k−1) — so the naive p-value stops lying (demo: 1.2e−5 → 0.19).023
Friedman testNon-parametric repeated-measures ANOVA on ranks: over N datasets, asks whether k models' average ranks differ more than chance. The omnibus step before a post-hoc (Demšar 2006).023
Nemenyi critical difference (CD)Post-hoc all-pairs test: two models differ significantly only if their average ranks differ by at least CD = q_α·√(k(k+1)/6N). Drawn as a CD diagram (a bar joins models that are not different).023
Random-search budget curveGrinsztajn's fair comparison (2022 §4): plot the expected test score of the best-validation config found after k random-search iterations, averaged over draw orderings. Shows both the default (k=1) and the tuned ceiling — not one number that hides tuning effort.024
Per-dataset (affine) normalizationRescale each dataset's scores so the worst model → 0 and the best → 1 ((s−worst)/(best−worst)) before averaging across a suite, because raw accuracies are incommensurable across datasets (echo of Demšar).024
Dataset-selection criteriaExplicit, pre-registered inclusion rules (real, tabular, medium-sized, heterogeneous columns, not too easy) applied to a large pool so a benchmark's verdict is not a dataset-selection artifact. Curation is half the experiment.024
Heterogeneous columnsEach column is a distinct, meaningful feature (not pixels or repeated sensor channels). The defining property of tabular data; homogeneous columns are perceptual data in disguise where neural nets have a natural edge.024
Default vs ceilingA model's untuned performance (budget 1) versus its best tuned performance (large budget). One tuned number collapses these into one; the budget curve keeps them separate — a strong default is cheaper to deploy.024
Smoothness (spectral) biasA neural net trained by gradient descent fits low-frequency (smooth) structure easily and high-frequency (jagged) detail slowly or not at all (Rahaman 2019). It can represent sharp functions, just not cheaply on finite data — so it over-smooths irregular tabular targets.025
Piecewise-constant fitA model that predicts a constant within each axis-aligned region (a decision tree). A single split makes an arbitrarily sharp step, so it can follow a jagged target — the opposite disposition to a smooth MLP.025
Target smoothing (Gaussian)Grinsztajn's §5.2 diagnostic: replace each target by a distance-weighted average of neighbours' targets, w_ij = exp(−‖x_i−x_j‖²/2h²). Growing h erases the target's high-frequency structure; the tree-vs-MLP gap collapses as that structure (variance) is removed — proving the smoothness bias is the mechanism. An ablation, never a production fix.025
Rotational invarianceA learning procedure is rotationally invariant (Ng 2004) if applying the same orthogonal matrix Q to the train and test features (X → X·Q) leaves its test performance unchanged. MLPs and ResNets are invariant (a linear first layer absorbs the rotation: W·(Qx)=(WQ)·x); decision trees are not (axis-aligned splits are tied to the original basis), and neither are FT-Transformers (pointwise tokenizer).026
Natural basisThe original feature axes of a table, where each column is individually meaningful (age, balance, store_id). Because tabular signal is aligned with this basis, a random rotation — which mixes columns into arbitrary combinations — destroys the alignment; unlike images, tabular data has a privileged basis, so rotation invariance is a liability, not a virtue.026
Random rotation (feature basis)Grinsztajn's §5.4 diagnostic: draw a random orthogonal Q (e.g. from a QR decomposition) and apply it to train + test features. The transform is lossless (Q is invertible), so any accuracy change isolates a model's bias: it collapses axis-aligned trees but leaves invariant MLPs unmoved, and can reverse the model ranking.026
Ng's sample-complexity boundNg (2004): any rotationally-invariant learning procedure has a worst-case sample complexity that grows at least linearly in the number of uninformative features — to drop a junk column it must first recover the original orientation, which rotation erases. The theoretical bridge from rotation invariance (L026) to fragility under uninformative features (L027).026
Uninformative (junk) featureA column statistically independent of the label (in experiments, a pure-noise Gaussian). Real tables are full of near-junk columns (surrogate IDs, audit fields, unrelated measurements). Grinsztajn §5.3 (Finding 2): MLP-like models are not robust to them; trees are.027
Implicit feature selectionA tree grows greedily, splitting only on the highest-gain feature, so a ~0-gain junk column is never chosen near the top — the tree selects the useful features as a side-effect of how it grows, with no separate selection step. The mechanism behind its robustness to junk; an MLP has no equivalent gate.027
Split gain (gate)The impurity a candidate split removes: gain = parent_impurity − weighted children impurity. A tree keeps only the highest-gain split, so a noise column (whose best split leaves the children as mixed as the parent, gain ≈ 0) loses to a real feature. Root-split gain was ~118× higher on informative than junk features — the gate that gives implicit feature selection.027
Root-split gain (vs MDI)The gain of a feature's best single threshold on the full data — where the tree's gate is real (junk ≈ 0). Preferred over impurity-based (MDI) importance for showing the gate, because deep in a tree spurious high-gain noise splits appear on small subsets, so MDI over-credits noise features.027
MLP (multilayer perceptron)A stack of linear (fully-connected) layers with a non-linearity (ReLU) between them: Linear → ReLU → … → Linear. Gorishniy's tabular MLP block is Dropout(ReLU(Linear(x))). The simplest neural baseline; still carries the L025–L027 inductive-bias debts.028
Residual / skip connectionA block that outputs x + f(x) instead of f(x): the input is added back through a shortcut that bypasses the layers, so the branch f only learns a small correction. Introduced by He et al. 2015; it makes the identity map free, so stacking blocks can't hurt.028
Degradation problemHe et al. 2015: a plain deep network gets worse as it gets deeper — and its training accuracy falls too, which rules out overfitting. It is an optimization failure (the extra layers can't easily learn the identity), distinct from vanishing gradients (which BatchNorm already fixes). The problem a skip connection solves.028
Tabular ResNetGorishniy 2021's residual net for tables: Linear_embed → [ResNetBlock]×N → Head, with ResNetBlock(x)=x+Dropout(Linear(Dropout(ReLU(Linear(BatchNorm(x)))))) and Head=Linear(ReLU(BatchNorm(x))). One of the two honest neural baselines a tabular-DL (or RDL) result must beat.028
BatchNormBatch normalization: per feature, subtract the minibatch mean and divide by its standard deviation, then rescale by two learned parameters. Stabilizes activation scale so training is faster and less sensitive to initialization/learning rate; largely fixes vanishing gradients (which is why the degradation problem is a separate issue).028
DropoutA regularizer that randomly zeroes a fraction p of activations during training (rescaling the rest) and is turned off at test time, so the network can't lean too hard on any one unit. Reduces overfitting; unrelated to the depth/degradation problem.028
Honest (neural) baselineA properly-tuned simple model used as the comparison bar. Gorishniy 2021's methodological finding: many "SOTA" tabular-DL papers failed to beat a tuned MLP/ResNet, so a claimed win against a weak baseline proves little. The single-table bar an RDL result must clear is a tuned GBDT and a tuned ResNet.028
No universal winnerGorishniy 2021's headline conclusion: across their benchmark, GBDT wins some datasets and DL wins others (ensembling both helps) — there is no category winner. On small categorical data (credit_g) the tuned GBDT still leads; the two neural baselines tie.028
AutoMLAutomated machine learning: a system that takes a raw dataset + a metric and produces a fitted model, choosing the algorithm, hyperparameters, and (some) preprocessing automatically. Canonical system: Auto-sklearn (Feurer 2015). It automates search, not domain feature engineering, and does not change the data representation.029
CASHCombined Algorithm Selection and Hyperparameter optimization (Thornton 2013, Auto-WEKA): treat "which algorithm" as a top-level categorical hyperparameter sitting above each model's own knobs, and search the whole nested space at once — A*, λ* = argmin_(A,λ) ValidationLoss(A(λ)). The core of what AutoML automates.029
Bayesian optimization (SMAC)The search strategy AutoML uses instead of grid/random: fit a cheap surrogate (Auto-sklearn uses a random forest, SMAC) that predicts a config's validation score + uncertainty, then pick the next config by trading exploitation vs exploration (an acquisition function). More sample-efficient than random search per trial.029
Meta-learning / warm-startLearning across datasets: describe each dataset by cheap meta-features (rows, features, imbalance, skew…), record which configs won on ~140 reference datasets, and warm-start a new search from the winners of the most similar past datasets. Auto-sklearn's first extension over Auto-WEKA; changes where the search begins, not what it searches.029
Ensemble selection (Caruana 2004)Auto-sklearn's second extension: build a weighted ensemble greedily from the pool of already-trained models — start empty, repeatedly add the model that most improves validation AUC, with replacement (so strong models get more weight). Leak-free (selected on validation), no meta-learner to overfit — the simpler cousin of L018 stacking. Gave +0.007 on credit_g.029
Benchmark reportThe one-page artifact a skeptic reads before believing "model A beats model B". A defensible report is fair (curated dataset, deployment-matched split, one metric + prevalence, disclosed budget, per-fold preprocessing), audited (a model info sheet on every feature's provenance), proven (a corrected significance test + effect size, and a budget curve rather than one tuned number), and explained (the ranking attributed to the three inductive biases) — ending in an honest verdict, including "no significant winner". The exact form in which every RDL claim must eventually be argued.030
Model info sheetKapoor & Narayanan's leakage checklist: before publishing, answer one question per leak type — most importantly, is each feature knowable strictly before the label's timestamp, and is any feature illegitimate/target-derived. Filling it (not assuming it) is what makes a benchmark report's feature set defensible.022, 030
Statistical tie (no significant winner)The honest verdict when the effect size, tested correctly, cannot be distinguished from noise. A reported gap is a random variable and k-fold scores overlap, so a bigger mean is not a win: on credit_g the GBDT led the neural baseline by +0.0081 ROC-AUC over 25 paired folds but the corrected resampled t-test gave p=0.64 → a tie. Being willing to report a tie is what makes an eventual "RDL wins" verdict from the same protocol credible.030

Q4 · Consolidation & bridge to neural tabular (L031–L040)

TermDefinitionLesson
Categorical encodingTurning a categorical column (labels from a finite set) into the numbers a model can consume. Four ways with different trade-offs: one-hot, ordinal, target, entity embedding. The choice is a representation decision — the axis Q4 opens after Q3 exhausted search and architecture.031
CardinalityThe number of distinct levels a categorical has (checking_status: 4; a zip_code/product_id: thousands). High cardinality is what makes the encoding choice matter — and is the property of the foreign-key IDs the relational thesis cares about.031
One-hot encodingA level of a k-level column becomes k binary columns (one 1, rest 0). Asserts nothing false (all levels equidistant) but is wide (+k columns per categorical; +54 on credit_g) and captures no similarity. Safe from leakage; infeasible at very high cardinality.031
Ordinal (label) encodingEach level → an integer (0,1,2,…): one compact column, but integers carry an order and spacing, so it invents a false order. A linear/neural model must honour it (credit_g: linear AUC 0.782→0.739); a tree splits around it (0.778→0.774), which is why GBDT libraries accept integer-coded categoricals.031
Out-of-fold (OOF) target encodingThe leak-free recipe: encode each fold's rows using level means computed only from the other folds, never a row's own label, and fit the encoder inside the CV fold (same discipline as L003/L005). Restored a signal-free id from a leaked 0.891 AUC to 0.504 (chance).031
Target-encoding smoothing (shrinkage)Shrink each level's mean toward the global prior by its count: enc(L) = (n_L·mean_L + m·global)/(n_L + m). A level with many rows keeps its mean; a rare level falls back to the prior, so it can't memorise a handful of rows. Distinct from the boosting learning-rate "shrinkage" (L013).031
Entity embeddingEach level → a short learned dense vector (dim ≈ min(50,(card+1)//2)), trained by backprop jointly with the model (Guo & Berkhahn 2016). Levels that behave alike end up close together — a similarity structure one-hot and ordinal lack; target encoding is the 1-D special case. The first learned representation and the atom of RDL (embedding foreign-key entities).031
Embedding dimensionThe width d of an entity-embedding vector. The Guo & Berkhahn / fast.ai heuristic is d = min(50, (cardinality+1)//2): far narrower than one-hot's k at high cardinality, and a knob trading capacity against overfitting on rare levels.031
Learned representationA numeric form of the raw data that is learned from the objective rather than hand-specified (an entity embedding vs one-hot). Q3 showed single-table search and architecture are exhausted; the learned representation is the open axis — and on a small flat table it ties, because there is little structure to learn (the payoff is at scale / on relational structure).031
Self-attentionThe core Transformer operation. Each input vector (token) emits a query, key and value (learned linear projections); a token's output is a weighted average of all tokens' values, with weights softmax(Q·Kᵀ/√d) (≥ 0, summing to 1). "Self" = the tokens attend to each other. In TabTransformer the tokens are a row's categorical columns.032
Contextual embeddingAn embedding whose vector depends on the rest of the input, unlike a context-free entity embedding (L031) that is the same everywhere. TabTransformer makes each categorical column contextual w.r.t. the other columns in the same row (the "bank" = river-bank vs savings-bank analogy) by passing the embeddings through self-attention.032
TabTransformerHuang et al. 2020 architecture: categorical entity embeddings → a stack of N Transformer self-attention layers → contextual embeddings; continuous features bypass the Transformer (LayerNorm only) and are concatenated with the flattened contextual vectors before an MLP head. Honest verdict: matches tree ensembles on supervised tabular (+1.0% is over other DL); real wins are robustness to noise/missingness and a +2.1% semi-supervised lift. Only categoricals are contextualised (FT-Transformer, Y2, tokenises numerics too).032
Thesis vocabulary (used from Year 1, defined in full later). Relational deep learning (RDL) — learning directly over a database's entity graph rather than a flattened table. Relational entity graph (REG) — the graph whose nodes are rows and edges are foreign keys. Message passing — how a GNN propagates information along those edges. These get their own entries when their lessons land (Year 3+); listed here so usage stays consistent from the start.
