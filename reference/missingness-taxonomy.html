Missingness: MCAR / MAR / MNAR

Reference · print-friendly cheat sheet

The one question

What does the probability of being missing depend on? Nothing → MCAR. Observed data → MAR. The missing value itself → MNAR.

Taxonomy

MechanismDepends onExample (weighing scale)
MCAR — completely at randomnothingloose wire drops readings at random
MAR — at randomobserved columnsmore drops on a soft surface (surface recorded)
MNAR — not at randomthe unobserved valuescale wears out, drops heavy objects (weight unknown)

Naming trap: MAR is not "random" — that's MCAR. MAR = random after conditioning on observed data.

What each handling assumes

MethodUnbiased underNote
Complete-case (drop rows)MCAR onlyunder MAR/MNAR you delete a biased slice
Mean / median imputation—shrinks variance, flattens relationships even under MCAR
Conditional (IterativeImputer, MICE)MCAR, MARneeds an observed predictor; fails for MNAR
Missing-indicator columnanykeeps "was-missing" signal (key for MNAR); doesn't restore the value
Native NaN (XGBoost/LightGBM)anylearns a default split direction = uses missingness as a feature

Testability

MCAR-vs-not is partly testable (compare observed distributions across the missing/observed split). MAR vs MNAR is untestable from the data — the needed info is the missing data itself. Argue the mechanism from how the data was generated; state it explicitly.

sklearn tools (all go inside the Pipeline)

from sklearn.impute import SimpleImputer, MissingIndicator
from sklearn.experimental import enable_iterative_imputer  # must precede next import
from sklearn.impute import IterativeImputer

SimpleImputer(strategy="median", add_indicator=True)   # fill + keep was-missing flag
IterativeImputer(max_iter=10, random_state=0)          # conditional, MAR-friendly

from sklearn.pipeline import Pipeline
pipe = Pipeline([
    ("impute", SimpleImputer(strategy="median", add_indicator=True)),
    ("clf",    LogisticRegression(max_iter=1000)),
])   # refit per fold (Lesson 005) — no leakage, MNAR signal kept

Relational / thesis notes

  • Relational gaps (no orders, null FK, no reviews) are usually structural, MNAR-flavoured — the absence depends on the target.
  • GBDT baseline: let it eat NaN natively; don't impute away a signal it could split on.
  • RDL: "missing" = an absent edge / neighbour in the entity graph; no NaN ever reaches the model.
  • Fair baseline rule: name the mechanism, keep an indicator, never silently drop rows.

Decision flow

Is a value missing?
 ├─ Does P(missing) depend on anything?  ── no ──▶ MCAR  (drop OR impute, both ok-ish)
 └─ yes
     ├─ only on OBSERVED columns? ──────── yes ─▶ MAR   (conditional impute; keep rows)
     └─ on the UNOBSERVED value? ───────── yes ─▶ MNAR  (indicator / native NaN; sensitivity-check)
